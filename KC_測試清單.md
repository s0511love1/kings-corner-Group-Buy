# King's Corner 測試清單 v2.1

> 測試前準備：
> - **電腦**：Chrome 開兩個分頁，一個 admin、一個 index（GitHub Pages）
> - **手機**：Safari（iOS）或 Chrome（Android）開啟 index GitHub Pages 網址

---

## 模組 A：商品庫

### A1 CSV 匯入
- [ ] 商品庫 → 匯入 CSV → 上傳測試檔
- [ ] 廠商不存在時彈出確認提示「將自動建立」
- [ ] 匯入 → 載入畫面顯示「第 X/Y 批」進度 → 完成

### A2 批次停用/啟用
- [ ] 勾選商品 → 一鍵停用 → 篩選切換到「全部（含停用）」，商品灰色可見
- [ ] 勾選停用商品 → 一鍵啟用 → 商品恢復正常

---

## 模組 B：優惠碼管理

### B1 新增優惠碼（多規則）
- [ ] 新增優惠碼，填代碼（TEST86）、名稱
- [ ] 「＋ 新增規則」→ 選初雞團購 → 每份折 5 元
- [ ] 再加一條 → 選逢國團購 → 百分比折扣 90
- [ ] 儲存 → Sheets 的 `campaignRules` 欄有 JSON（不是時間）
- [ ] 列表顯示「2 條規則」及折扣摘要

### B2 編輯後規則保留
- [ ] 點「編輯」→ 兩條規則正確顯示
- [ ] 修改其中一條 → 儲存 → 再次編輯確認已更新

### B3 刪除優惠碼
- [ ] 點「刪除」→ 確認 → 列表**立即消失**（不需重整）

---

## 模組 C：開團與連結

### C1 新增團購
- [ ] 填設定 → 開放優惠碼勾選 TEST86
- [ ] 儲存 → **2 秒後自動關閉**

### C2 Modal 防誤觸
- [ ] 開啟新增團購視窗 → 點視窗外 → **視窗不關閉**

### C3 批次狀態
- [ ] 勾選進行中 → 設為草稿 → index 重整後消失
- [ ] 篩選草稿 → 設為進行中 → index 重整後出現

### C4 再次開團
- [ ] 歷史紀錄 → 「🔄 再次開團」→ 舊設定帶入，截止日空白
- [ ] 填新截止日 → 儲存 → 產生新團購

---

## 模組 D：購物車下單（電腦版）

### D1 首頁
- [ ] 開啟 index → 預設顯示「全部」所有進行中的團購（不自動選廠商）
- [ ] 點廠商分頁 → 正確篩選；點「全部」恢復

### D2 加入購物車基本流程
- [ ] 點「🛒 加入購物車」→ 進入商品選擇頁
- [ ] 數量欄位可直接輸入數字，也可按 + / − 調整
- [ ] 輸入 TEST86 → 點「套用」→ 顯示折扣說明
- [ ] 點「加入購物車」→ 「✅ 已加入購物車！」→ 自動回首頁
- [ ] 右下角購物車圖示出現數量角標

### D3 重新進入修改（關鍵）
- [ ] 再次點同一個團購「加入購物車」
- [ ] 自動帶入上次已選的**數量**（不是清零）
- [ ] 自動帶入上次輸入的**優惠碼**
- [ ] 按鈕顯示「✅ 更新購物車」
- [ ] 修改數量 → 更新 → 「✅ 購物車已更新！」

### D4 優惠碼 Race Condition 測試
- [ ] 輸入 TEST86 **不按套用**，直接點「加入購物車」
- [ ] 系統自動驗證再加入
- [ ] 購物車裡的折扣正確計入（不是原價）

### D5 多廠商購物車
- [ ] 加入兩個不同廠商的商品
- [ ] 購物車頂部輸入 TEST86 → 套用
   - 初雞顯示「每份折 $5」
   - 逢國顯示「90折」
- [ ] 底部明細各廠商都有小計行
- [ ] × 清除優惠碼 → 兩廠商恢復原價

### D6 結帳流程
- [ ] 選自取 → 顯示「匯款」和「現金」兩個付款按鈕
- [ ] 選 7-11 → 現金按鈕消失，地址欄出現
- [ ] 底部明細：商品 + 折扣 + 小計 + 運費 + 合計
- [ ] 備註區各廠商金色標題分隔顯示
- [ ] **未勾選備註確認** → 按送出 → 滾動到勾選框，**購物車資料保留**
- [ ] 勾選後送出 → 成功頁顯示訂單編號和取貨方式
- [ ] 成功頁顯示「截止日前未完成付款，訂單視同不成立」

### D7 送出結果驗證
- [ ] Admin 出現**兩筆獨立訂單**（各廠商各一筆）
- [ ] Sheets 訂單工作表有 shippingMethod、paymentMethod 欄位

### D8 防重複下單
- [ ] 同手機號碼、同團購再次下單 → 系統拒絕

## 模組 D-M：購物車下單（手機專項）

> 建議使用 iOS Safari 測試，這是最嚴格的環境

### DM1 基本顯示
- [ ] 首頁所有團購卡片完整顯示，不被裁切
- [ ] 廠商分頁標籤可水平捲動（若廠商多）
- [ ] 「🛒 加入購物車」按鈕觸控區夠大，不需精確點擊

### DM2 縮放問題
- [ ] 點 + / − 按鈕**不會觸發畫面縮放**
- [ ] 快速連點數量按鈕不會縮放或跑版

### DM3 鍵盤行為
- [ ] 點數量輸入框 → 彈出**數字鍵盤**（不是英文鍵盤）
- [ ] 點優惠碼輸入框 → 彈出英文鍵盤，輸入自動轉大寫
- [ ] 鍵盤彈出後頁面不跑版，輸入框仍可見

### DM4 購物車頁面
- [ ] 購物車圖示不被 iOS Safari 底部工具列遮住
- [ ] 購物車頁面可以正常上下捲動（不卡住）
- [ ] 各廠商區塊、備註區、選項區均可捲動到
- [ ] 「確認送出訂單」按鈕不被底部工具列遮住

### DM5 返回與導航
- [ ] 商品選擇頁點「← 回首頁」→ 首頁正常顯示所有團購
- [ ] 購物車頁點「←」→ 首頁正常顯示（不只剩 Banner）
- [ ] 瀏覽器上滑返回手勢不觸發意外行為

### DM6 觸控體驗
- [ ] 優惠碼「套用」按鈕觸控區夠大（建議 44px 以上）
- [ ] 「× 清除」按鈕可輕鬆點到（不需精確）
- [ ] 備註確認勾選框觸控區夠大，不難勾選
- [ ] 取貨方式三個按鈕均可點選，選中後有明顯視覺回饋
