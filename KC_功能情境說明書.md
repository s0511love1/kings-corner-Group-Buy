> 最後更新：2026-06-25

## 角色說明

| 角色 | 說明 |
|---|---|
| 平台管理員 | ROOT 帳號，所有功能 |
| 員工 | ADMIN / HELPER / VIEWER，依角色限制 |

## 情境一：開啟新團購

1. Admin → 🛒 團購管理 → ＋ 新增團購
2. **第一步：基本設定**
   - 填名稱、選廠商、截止日期、最低成團人數、折扣
   - 備註/付款說明：手填或點「套用範本▾」帶入
   - 預計到貨日：開團時即可填入（格式 MM/DD，例：07/15），客人在首頁卡片即可看到
   - 開放優惠碼：勾選本次允許使用的優惠碼（白名單制）
3. **第二步：選擇品項** — 勾選商品，可依分類搜尋
4. **第三步：確認品項/單價/庫存** → 儲存 → 2 秒後自動關閉
5. 在列表點「連結」→ 複製一般連結貼到 LINE 群組

---

## 情境二：設定優惠碼與產生團購主專屬連結

1. 🎟️ 優惠碼管理 → ＋ 新增優惠碼
2. 填優惠碼（自動查重，不分大小寫）、團購主名稱、聯絡方式
3. 點「＋ 新增規則」→ 選對應團購 → 選折扣類型與值：
   - **百分比折扣**：覆蓋團購折扣（例：83折）
   - **每份折固定金額**：每份商品扣固定金額（例：每份 -$5）
   - **滿額折扣**：達門檻後折固定金額（例：滿$1000折$100）
   - **純追蹤**：不改變價格，只記錄使用次數
4. 未設定規則的團購預設「不允許使用」
5. 儲存後，去團購管理編輯對應團購，「開放優惠碼」勾選此碼
6. 開團 Wizard 第三步 → 選擇優惠碼 → 「產生」→ 複製專屬連結
   - 格式：`...?cid=C123&code=BOB88`

## 情境三：客人購物車下單完整流程

### 加入購物車

1. 開啟 index 首頁（預設顯示「全部」進行中的團購）
2. 點廠商分頁標籤可依廠商篩選
3. 點「🛒 加入購物車」→ 進入商品選擇頁
4. **商品選擇頁：**
   - 輸入優惠碼（選填）→ 點「套用」試算折扣
   - ※ 每筆訂單限使用一組，可套用多個團購
   - 調整商品數量（可直接輸入數字，或用 + / − 按鈕）
   - 底部即時顯示金額與省多少錢
   - 點「加入購物車」→ 系統若偵測到優惠碼尚未驗證，自動先驗證再加入
   - 頂部彈出「✅ 已加入購物車！」→ 自動回首頁

### 修改購物車內容

- 再次點同一個團購的「加入購物車」→ 進入商品選擇頁
- **自動帶入上次已選的數量和優惠碼**
- 修改後點「✅ 更新購物車」→ 顯示「✅ 購物車已更新！」

### 結帳

1. 右下角購物車圖示（顯示商品數量）→ 點開購物車結帳頁
2. 頂部輸入優惠碼（一次套用到所有廠商）
   - 各廠商區塊顯示對應折扣結果
   - × 清除鍵可移除優惠碼，恢復原價
3. 選取貨方式：自取 $0 / 7-11 $65 / 黑貓 $170
4. 選付款方式：匯款 / 現金（現金限自取）
5. 填姓名、手機（宅配需填地址）
6. 備註/付款說明彙整顯示（金色標題區分各廠商）
7. 勾選「已閱讀並了解備註 / 付款說明」→ 按鈕啟用
8. 點「確認送出訂單」→ 各廠商各自產生獨立訂單
9. 成功畫面顯示訂單編號、取貨方式
   - 顯示「請於截止日前完成付款，截止日前未完成付款訂單視同不成立」

> **注意：** 未勾選備註確認就按送出，購物車資料保留，補勾後可直接重送，不需重填。

## 情境四：優惠碼使用邏輯

**一個優惠碼對應一位團購主，可對不同團購設不同規則：**
```
BOB88（Bob Wang）：
  初雞 A 團購 → 每份 -$5
  逢國 B 團購 → 9折
  其他團購     → 不允許使用（預設）
```

客人輸入 BOB88，系統自動對購物車每個廠商驗證：
- 初雞 → 顯示「每份折 $5」
- 逢國 → 顯示「9折」
- 未設定規則的廠商 → 無折扣

**白名單：** 開團時必須勾選允許的優惠碼，未勾選則無效。

## 情境五：批次管理團購狀態

1. 團購管理 → 勾選目標團購
2. 操作列：
   - **▶ 設為進行中** — 客人可看到並下單
   - **⏸ 設為草稿/暫停** — 隱藏，保留資料，可隨時恢復
   - **⏹ 設為已結束** — 移至歷史紀錄（需確認）

---

## 情境六：結單與匯出已付款清單

1. 付款追蹤 → 篩選廠商 + 團購
2. 勾選已付款訂單 → 「✓ 一鍵標記已付款」
3. 確認篩選為已付款 → 「⬇ 匯出已付款 Excel」
   - 檔名：`初雞202607團購_已付款_20260622.xlsx`
   - 欄位：姓名、手機、品項、數量、金額、合計

---

## 情境七：再次開團（套用舊設定）

1. 歷史紀錄 → 歷史團購 → 「🔄 再次開團」
2. 自動帶入所有舊設定，截止日期清空必須重填
3. 名稱加「(複製)」提醒修改
4. 儲存 → 產生全新獨立團購

---

## 情境八：批次管理商品庫

**CSV 匯入：**
- 欄位：廠商名稱、分類、名稱、規格、建議單價、圖片網址
- 廠商不存在時自動建立
- 系統分批 20 筆 POST，大量商品也不失敗

**批次停用/啟用：**
勾選商品 → 一鍵停用（停用後自動切換到「全部含停用」，仍可見並可重新啟用）

## 情境九：備註範本管理

**儲存：** 備註欄填寫 → 「儲存為範本」→ 輸入名稱（上限 5 個）

**套用：** 「套用範本▾」→ 選擇 → 自動填入

上限後需至 Sheets「行銷設定」刪除 `note_template_X`

## 情境十：優惠碼使用統計

1. 🎟️ 優惠碼管理 → 「📊 使用統計」→ 設定日期區間 → 查詢
2. 顯示各優惠碼在各團購的使用次數與累計金額

## 情境十一：客人查詢訂單

1. Index 首頁 → 「📋 查詢我的訂單」
2. 輸入手機號碼 → 顯示所有訂單、品項、付款狀態

---

## 情境十二：商品圖片批次管理

**廠商提供圖片：**
1. 給廠商 `KC_廠商圖片範本.md`，請廠商填寫：商品名稱（須與商品庫完全一致）+ Google Drive 分享連結
2. 收到後，把資料貼到 Sheets「圖片清單」工作表（B欄=商品名稱、C欄=網址）
3. GAS 執行 `autoFillProductImages()` → 自動轉換網址格式 + 嚴格比對寫入商品庫
4. 執行記錄列出未配對的商品 → 修正名稱後重新執行

**注意：** 商品名稱比對為嚴格完全比對，一個字不能差，避免類似商品配錯圖。

---

## 情境十三：已結單團購進度更新

**Admin 快速更新（不用進編輯視窗）：**
1. 團購管理 → 篩選「已結束（含結單進度）」
2. 每列狀態欄直接有下拉選單 → 選擇新狀態 → 自動儲存
   - 🕐 結單中 → 🚚 出貨中 → 📦 已到貨

**客人端顯示：** Index 首頁「已結單團購處理進度」區塊即時反映狀態與預計到貨日

**自動關閉到期團購：**
執行一次 `createAutoCloseTrigger()` 後，每天凌晨 1 點自動把截止日已過的 `active` 團購改為 `closed`
